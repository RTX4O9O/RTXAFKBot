# AGENTS.md — Guide for AI coding agents

Purpose: help automated agents quickly understand, modify, and verify the FakePlayerPlugin codebase.

1) Quick repo map
- `src/main/java/me/bill/fakePlayerPlugin/` — main code and entry points
- `src/main/resources/` — default `config.yml`, `plugin.yml`, `bot-names.yml`, `bot-messages.yml`, `secrets.yml` (AI API keys template — copied to `plugins/FakePlayerPlugin/secrets.yml` on first run; managed by `SecretsConfig`). Bundled sample personality prompts: `personalities/friendly.txt`, `personalities/grumpy.txt`, `personalities/noob.txt` — extracted to `plugins/FakePlayerPlugin/personalities/` on first run.
- `src/main/resources/language/en.yml` — all user-facing messages (loaded by `Lang.java`)
- `pom.xml` — build + jar metadata (`target/fpp-*.jar` produced)
- `frontend/`, root `package.json`, and `vercel.json` — separate Node/Vercel status + update API sidecar used by the plugin’s update-check fallback; not part of the Maven plugin jar build.
- `frontend/wiki/` — primary feature docs (Commands.md, Configuration.md, Getting-Started.md, Database.md, Language.md, Permissions.md, Skin-System.md, Swap-System.md, Bot-Behaviour.md, Bot-Messages.md, Bot-Names.md, Fake-Chat.md, FAQ.md, Migration.md, Home.md, Proxy-Support.md, Config-Sync.md, Placeholders.md, Peak-Hours.md, Changelog.md). Root `wiki/` currently contains a small subset (e.g. Changelog/Proxy/Config-Sync/Placeholders).
- `src/main/java/me/bill/fakePlayerPlugin/gui/` — GUI systems: `SettingGui` (in-game settings, `/fpp settings`), `HelpGui` (paginated command GUI used by `/fpp help`; console still uses text fallback), and `BotSettingGui` (per-bot 6-row chest GUI; opened by shift-right-clicking a bot entity when `Config.isBotShiftRightClickSettingsEnabled()` — categories: General, Chat, PvP, Cmds, Danger; uses `BotRenameHelper` for in-GUI rename; instantiated in `onEnable`, registered as listener, passed to `InventoryCommand` constructor).
- `src/main/java/me/bill/fakePlayerPlugin/util/` — helper subsystems and utilities (e.g. `FppMetrics`, `FppLogger`, `FppPlaceholderExpansion`, `TabListManager`, `BackupManager`, `UpdateChecker`, `ConfigValidator`, `DataMigrator`, `TextUtil`, `ConfigMigrator`, `LuckPermsHelper`, `CompatibilityChecker`, `YamlFileSyncer`, `BotTabTeam`, `WorldGuardHelper`, `BadwordFilter`, `BotRenameHelper`). `BotRenameHelper` centralises the validate → delete → respawn → restore lifecycle shared by `RenameCommand` and `BotSettingGui`.
- `src/main/java/me/bill/fakePlayerPlugin/listener/` — Bukkit event listeners (`PlayerJoinListener`, `FakePlayerEntityListener`, `FakePlayerKickListener`, `BotCollisionListener`, `PlayerWorldChangeListener`, `ServerListListener`, `BotCommandBlocker`, `BotSpawnProtectionListener`, `BotXpPickupListener`, `BotMessageListener`). Note: `LuckPermsUpdateListener` was removed — LP now detects NMS bots as real players automatically. `BotCommandBlocker` prevents bots from executing any commands. `BotSpawnProtectionListener` prevents lobby/spawn plugins from teleporting bots during their 5-tick spawn grace period. `BotXpPickupListener` gates XP orb pickup (`PlayerPickupExperienceEvent`) and programmatic XP grants (`PlayerExpChangeEvent`) for bots via `Config.bodyPickUpXp()` and the `XpCommand` post-collection cooldown. `BotMessageListener` intercepts `/msg`, `/tell`, `/whisper` (and aliases `/w`, `/m`, `/t`, `/pm`, `/dm`, etc.) directed at bots, cancels the event, and routes to `BotConversationManager`; registered only when `Config.aiConversationsEnabled() && aiProviderRegistry.isAvailable()`.
- `src/main/java/me/bill/fakePlayerPlugin/fakeplayer/` — bot domain objects and subsystems (e.g. `FakePlayer`, `FakePlayerBody`, `FakePlayerManager`, `BotPersistence`, `ChunkLoader`, `SkinRepository`, `SkinFetcher`, `SkinProfile`, `SkinProfileInjector`, `NmsHelper`, `NmsPlayerSpawner`, `NmsReflection`, `BotSwapAI`, `BotChatAI`, `BotBroadcast`, `PacketHelper`, `RemoteBotCache`, `RemoteBotEntry`, `BotType`, `BotCombatMode`, `BotPvpAI`, `PeakHoursManager`, `BotPathfinder`, `PathfindingService`, `BotNavUtil`, `StorageInteractionHelper`). **`PathfindingService`** centralises all navigation loops previously duplicated across `MoveCommand`, `MineCommand`, `PlaceCommand`, and `UseCommand`. `Owner` enum: `MOVE`, `MINE`, `PLACE`, `USE`, `SYSTEM`. `NavigationRequest` record: `owner`, `destinationSupplier`, `arrivalDistance`, `recalcDistance`, `maxNullPathRecalculations`, `onArrive`, `onCancel`, `onPathFailure`, `lockOnArrival` (optional — when non-null, `cleanup()` atomically calls `manager.lockForAction(botUuid, lockOnArrival)` before firing `onArrive`, eliminating the one-tick gap between nav-lock release and action-lock acquisition). Backward-compatible 8-arg constructor sets `lockOnArrival = null`. Public API: `navigate(fp, request)`, `cancel(uuid)`, `cancelAll()`, `cancelAll(owner)`, `isNavigating(uuid)`, `isNavigating(uuid, owner)`, `getOwner(uuid)`; exposed via `plugin.getPathfindingService()`. **`BotNavUtil`** — static utility class consolidating navigation/interaction helpers shared by `MineCommand` and `PlaceCommand`: `findStandLocation(World, SelectionBounds, tx, ty, tz)` (16-candidate walkable-adjacent search, prefers positions outside the selection), `faceToward(from, target)` (computes yaw+pitch toward a location), `isAtActionLocation(bot, loc)` (XZ ≤ 0.35 proximity check), `useStorageBlock(bot, block)` (NMS right-click via `gameMode.useItemOn`). `SelectionBounds` is an inner `@FunctionalInterface` (`boolean contains(int x, int y, int z)`) so callers pass `sel::contains` without coupling to command internals. **`StorageInteractionHelper`** — encapsulates the lock→open-container→transfer→unlock lifecycle shared by `MineCommand` (deposit bot→storage) and `PlaceCommand` (fetch storage→bot). Static entry point: `interact(fp, faceLoc, block, plugin, manager, transferFn, onFinally)`. Timing: bot locked + teleported on call; +1 tick container opened via NMS; +3 ticks transfer runs, inventory closed, action lock released, GUI refreshed, `onFinally` called. All error paths (bot offline, block no longer a container) also call `onFinally` so callers can clean up gating flags (e.g. `job.depositingToStorage = false`). **BotSwapAI rejoin lifecycle (v1.5.17 critical fix):** rejoin task is now registered AFTER `delete()` completes so `cancel(uuid)` never silently kills the rejoin timer — bots actually come back. New config: `swap.min-online` (floor that prevents swaps if removing one would drop below), `swap.retry-rejoin` (auto-retry failed rejoins, default `true`), `swap.retry-delay` (seconds before retry, default `60`). Bot identification on rejoin: same-name rejoins use `getByName()` (O(1) `nameIndex` lookup); random-name rejoins use UUID diff. Sub-package `fakeplayer/network/` contains the fake Netty layer: `FakeChannel` (discard channel used by `FakeConnection`), `FakeChannelPipeline` (no-op pipeline), `FakeConnection` (NMS `Connection` subclass — overrides all three `send()` variants as no-ops to prevent vanilla connection side-effects during `placeNewPlayer()` that previously caused a phantom "Anonymous User" / UUID-0 ghost entry; `isConnected()` always returns `true`; injects `FakeChannel` and calls `Connection.configureSerialization()` on the no-op pipeline to satisfy NMS internal checks), and `FakeServerGamePacketListenerImpl` (no-op SGPL subclass that discards all outbound packets except knockback). Note: `BotHeadAI` has been removed; head-AI functionality (player tracking, look-range, turn-speed) is now driven by config keys `head-ai.*` and executed inside `FakePlayerManager`.
- `src/main/java/me/bill/fakePlayerPlugin/messaging/` — proxy communication (`VelocityChannel`): registers `fpp:main` plugin-messaging channel; wraps outbound messages in a BungeeCord `Forward ALL` envelope so Velocity/BungeeCord re-delivers to every backend. Subchannels: `BOT_SPAWN`, `BOT_DESPAWN`, `BOT_UPDATE` (display-name sync — wire: `[msgId][serverId][uuid][newDisplayName]`; sent by `broadcastBotDisplayNameUpdate(fp)` when LP refreshes a bot's display name), `CHAT`, `ALERT`, `JOIN`, `LEAVE`, `SYNC`. Echo suppression via per-message unique IDs.
- `src/main/java/me/bill/fakePlayerPlugin/sync/` — proxy config sync (`ConfigSyncManager`): push/pull config files to/from MySQL in NETWORK mode. Supports `DISABLED`, `MANUAL`, `AUTO_PULL`, `AUTO_PUSH` modes. Server-specific keys (`database.server-id`, `database.mysql.*`, `debug`) are never synced.
- `src/main/java/me/bill/fakePlayerPlugin/database/` — session history and analytics (`DatabaseManager`, `BotRecord`). **Schema version 13** — persistent tables: `fpp_bot_sessions`, `fpp_active_bots` (v8→v9 added `frozen`, `chat_enabled`, `chat_tier`, `right_click_cmd`; v9→v10 added `ai_personality VARCHAR(64)`; v10→v11 added `pickup_items BOOLEAN`, `pickup_xp BOOLEAN`), `fpp_sleeping_bots` (peak-hours crash recovery), `fpp_bot_identities` (stable name→UUID registry), **`fpp_bot_tasks`** (v12→v13 — bot task persistence: mine/use/place/patrol state survives server restarts; one row per active task type per bot; columns: `bot_uuid`, `server_id`, `task_type` VARCHAR(16) — values `MINE`/`USE`/`PLACE`/`PATROL`, `world_name`, `pos_x/y/z`, `pos_yaw/pitch`, `once_flag`, `extra_str` VARCHAR(256) — patrol route name, `extra_bool` — patrol random flag; PRIMARY KEY `(bot_uuid, server_id, task_type)`), `fpp_meta`. New API: `saveBotTasks(List<BotTaskRow>)` (enqueued write — replaces all rows for this server then inserts fresh), `loadBotTasksForThisServer()` (synchronous read → `List<BotTaskRow>`), `clearBotTasks()` (enqueued delete — called after tasks are loaded to prevent double-restore), `updateBotAiPersonality(String botUuid, String personality)` (enqueued write). `BotTaskRow` record: `botUuid`, `serverId`, `taskType`, `worldName`, `posX/Y/Z`, `posYaw/Pitch`, `onceFlag`, `extraStr`, `extraBool`.
- `src/main/java/me/bill/fakePlayerPlugin/lang/` — message rendering (`Lang`).
- `src/main/java/me/bill/fakePlayerPlugin/permission/` — centralised permission registry (`Perm`).
- `src/main/java/me/bill/fakePlayerPlugin/ai/` — AI conversation system: `AIProvider` (interface with `ChatMessage` record), `AIProviderRegistry` (selects first available provider; `isAvailable()`, `getActiveProvider()`, `generateResponse()`), `BotConversationManager` (per-bot-player conversation history, DM handling, public-chat reactions, personality resolution, rate limiting; `handleDirectMessage`, `generatePublicChatReaction`, `clearAll()`), `PersonalityRepository` (loads `.txt` files from `plugins/FakePlayerPlugin/personalities/`; `get(name)`, `has(name)`, `getNames()`, `reload()`). Seven concrete providers: `OpenAIProvider`, `AnthropicProvider`, `GroqProvider`, `GoogleGeminiProvider`, `OllamaProvider`, `CopilotProvider`, `CustomOpenAIProvider`. API keys live in `plugins/FakePlayerPlugin/secrets.yml` (managed by `config/SecretsConfig.java`); template extracted from JAR on first run. Accessed via `plugin.getAIProviderRegistry()`, `plugin.getBotConversationManager()`, `plugin.getPersonalityRepository()`.
- `config/SecretsConfig.java` — manages `plugins/FakePlayerPlugin/secrets.yml`; exposes API key accessors for all seven AI providers (`openaiApiKey()`, `groqApiKey()`, etc.); `init(plugin)` copies template from JAR on first run; `reload()`. **Never commit real API keys** — the template file is the only version-controlled copy.

2) Where to start (entry points)
- `FakePlayerPlugin.java` — plugin lifecycle (`onEnable` / `onDisable`) and orchestration.
- `src/main/java/me/bill/fakePlayerPlugin/command/` — command classes (`SpawnCommand`, `DeleteCommand` (registers as `despawn`), `ListCommand`, `HelpCommand`, `InfoCommand`, `ChatCommand`, `SwapCommand`, `ReloadCommand`, `FreezeCommand`, `StatsCommand`, `MigrateCommand`, `TpCommand`, `TphCommand`, `LpInfoCommand`, `RankCommand`, `AlertCommand`, `SyncCommand`, `PeaksCommand`, `SettingCommand`, `InventoryCommand`, `MoveCommand`, `XpCommand`, `CmdCommand`, `MineCommand`, `UseCommand`, `WaypointCommand`, `BadwordCommand`, `RenameCommand`, `PersonalityCommand`, `PlaceCommand`, `StorageCommand`) plus `WaypointStore` (named route persistence used by `/fpp waypoint` and `/fpp move --wp`), `StorageStore` (per-bot supply-container registry, `data/storage-points.yml`), `MineSelectionStore` (area selection persistence for `/fpp mine`, `data/mine-selections.yml`).
- `command/FppCommand.java` — interface every sub-command implements (`getName`, `getAliases` (default `emptyList()` — used e.g. by `InventoryCommand` for alias `"inv"`), `getUsage`, `getDescription`, `getPermission`, `canUse`, `execute`, `tabComplete`).
- `config/Config.java`, `util/ConfigMigrator.java` — configuration pattern and automatic migration.
- `fakeplayer/FakePlayerManager.java`, `fakeplayer/BotPersistence.java` — bot lifecycle, persistence and restore. Key new helpers: `requestNavJump(UUID)` — sets a 5-tick jump-hold for navigation (used by `MoveCommand` to prevent the swim-AI from cancelling jumps before physics fires); `clearNavJump(UUID)` — clears the pending jump on despawn. **Action lock API (renamed from mining lock):** `lockForAction(UUID, Location)` — freezes the bot at the given position and suppresses head-AI for the duration of a block interaction (mining, storage deposit/fetch, right-click use); `unlockAction(UUID)` — releases the lock; `isActionLocked(UUID)` — query. Backed by `actionLockedBots` (`ConcurrentHashMap<UUID, Location>`). Previously named `lockForMining`/`unlockMining`/`isMiningLocked` — all old names have been removed.
- `fakeplayer/BotPathfinder.java` — server-side A* grid pathfinder for bot navigation (used by `MoveCommand`). Supports 6 `MoveType` actions: `WALK`, `ASCEND`, `DESCEND`, `PARKOUR`, `BREAK`, `PLACE`. Feature flags passed via `PathOptions` record (`parkour`, `breakBlocks`, `placeBlocks`); all default `false`. Key limits: `MAX_NODES=2000` (plain), `MAX_NODES_EXTENDED=4000` (with advanced options), `MAX_RANGE=64` blocks straight-line. Public entry points: `BotPathfinder.findPathMoves(world, sx,sy,sz, tx,ty,tz, opts)` → `List<Move>`; `BotPathfinder.findPath(...)` → `List<Pos>` (backward-compat, strips action metadata). `walkable(world, x, y, z)` and `passable(world, x, y, z)` are public world-query helpers.
- `command/InventoryCommand.java` — `/fpp inventory <bot>` (alias `inv`). Opens a 54-slot double-chest GUI displaying the bot's full inventory (main storage rows 1-3, hotbar row 4, label bar row 5, equipment+offhand row 6). Implements both `FppCommand` and `Listener`. Equipment slots enforce type restrictions (boots/leggings/chestplate-or-elytra/helmet/offhand). Right-clicking a bot entity directly opens the GUI without a command (unless shift-held, which opens `BotSettingGui` instead). `openGui(Player, FakePlayer)` is the public entry point. Constructor: `new InventoryCommand(fakePlayerManager, plugin, botSettingGui)`. Registered as a listener in `onEnable` immediately after command registration. Permission: `Perm.INVENTORY = "fpp.inventory"`.
- `command/MoveCommand.java` — `/fpp move <bot> <player>`. Navigates a bot to an online player using `PathfindingService` (navigation loop is no longer internal — all path execution delegated to `PathfindingService`). Recalculates path when target moves >3.5 blocks or every 60 ticks. Calls `manager.requestNavJump(UUID)` for ascend/parkour/stuck-recovery jumps. Stuck detection (8 ticks without movement) forces jump + recalc. Public `cleanupBot(UUID)` method removes bot's entry from `waypointSets` map to prevent memory leaks on bot despawn — called by `FakePlayerManager.delete()`. Supports `/fpp move <bot> --stop` to pause navigation without deleting waypoints. `cancelAll()` is a public method but is NOT called in `onDisable` (nav tasks cancel themselves on arrival or when the bot/target goes offline). Permission: `Perm.MOVE = "fpp.move"`.
- `command/XpCommand.java` — `/fpp xp <bot>`. Transfers all XP (`getTotalExperience()`) from a bot to the command sender (player only; console gets nothing but XP is still cleared). Clears bot's total XP, level, and progress. Imposes a 30-second post-collection cooldown on the bot's XP pickup (`BotXpPickupListener` reads `plugin.getXpCommand().isOnXpCooldown(uuid)`). Exposed via `plugin.getXpCommand()`. Permission: `Perm.USER_XP = "fpp.xp"` (user-tier, included in `fpp.use`).
- `command/CmdCommand.java` — `/fpp cmd <bot> <command...>` (direct execution), `/fpp cmd <bot> --add <command...>` (store right-click command), `/fpp cmd <bot> --clear` (remove stored command), `/fpp cmd <bot> --show` (display stored command). Uses `Bukkit.dispatchCommand()` which bypasses `PlayerCommandPreprocessEvent` so `BotCommandBlocker` does not interfere. Right-click behavior handled in `InventoryCommand.onRightClickBot()` — bots with stored commands execute them instead of opening inventory GUI. Permission: `Perm.CMD = "fpp.cmd"`.
- `command/MineCommand.java` — `/fpp mine <bot>` (continuous mining), `/fpp mine <bot> once` (mine one block), `/fpp mine <bot> stop` (stop mining), `/fpp mine stop` (stop all bots). Bots continuously break blocks they're looking at using NMS `handleBlockBreakAction()` and `destroyBlockProgress()` packets. Creative mode: instant break with 5-tick cooldown. Survival mode: progressive mining with destroy-speed accumulation. Mining state tracked per-bot (`MiningState` — `currentPos`, `progress`, `freeze`, `once`). 1-tick repeating task per active miner; tasks auto-cancel when bot/target goes offline. Area-selection mode: `/fpp mine <bot> --pos1`, `--pos2`, `--start`, `--status`, `--stop` — selects a cuboid region and mines it continuously; selections persist across restarts via `MineSelectionStore` (`data/mine-selections.yml`) and auto-resume. Constructor: `new MineCommand(plugin, fakePlayerManager, storageStore, mineSelectionStore, pathfindingService)`. Restocks from nearest registered `StorageStore` container when inventory fills. Public API: `stopMining(UUID)`, `stopAll()`. Permission: `Perm.MINE = "fpp.mine"`.
- `command/RenameCommand.java` — `/fpp rename <oldname> <newname>`. Renames an active bot by fully despawning the old entity and respawning it under the new name. Rename lifecycle is now delegated to `BotRenameHelper` (shared with `BotSettingGui`). Preserves all bot state via `BotSnapshot`: inventory (main/armor/offhand, deep-cloned), XP (level + progress + total), chat settings (enabled flag + activity tier), frozen state, LuckPerms group, `aiPersonality`, and original spawner info. **Rename lifecycle:** (1) snapshot captured from old FakePlayer, (2) `manager.markRenaming(oldUuid)` called to suppress join/leave broadcasts, (3) `manager.delete(oldName)` fires, (4) 12 ticks later `manager.spawn(loc, 1, null, newName, true, botType)` called, (5) new bot UUID also marked renaming, (6) `scheduleRestore` polls every 5 ticks until NMS entity ready (up to 200 ticks), (7) snapshot applied, LP group restored 5 ticks later, (8) both UUIDs unmarked, (9) `bot-rename` Lang broadcast sent to all players. The delete+respawn approach guarantees the nametag above the head and death messages reflect the new name (GameProfile name is baked into the NMS entity at spawn time and cannot be changed in-place). **Permission tiers:** `Perm.RENAME = "fpp.rename"` — rename any bot (admin, default: op); `Perm.RENAME_OWN = "fpp.rename.own"` — rename only bots the sender personally spawned (user opt-in, default: false). Tab-complete for arg[0] shows only bots the sender can rename. `fpp.rename` is a parent of `fpp.rename.own` in `plugin.yml` so admins inherit both. **Rename tracking:** `FakePlayerManager.renamingBotIds` (ConcurrentHashSet<UUID>) — both old and new UUIDs are in this set while the rename is in progress. `PlayerJoinListener.onJoin` checks `manager.isRenaming(fp.getUuid())` to suppress the join message for the new entity; `PlayerJoinListener.onQuit` checks `manager.isRenaming(uuid)` in the `isDespawning` branch to suppress the leave broadcast for the old entity. `FakePlayerManager.delete()` also gates the VelocityChannel proxy leave broadcast on `!renamingBotIds.contains(target.getUuid())`. The `bot-rename` Lang key (`{old}` / `{new}` placeholders) is broadcast once when restore completes.
- `fakeplayer/PeakHoursManager.java` — time-window-based bot pool scheduler. Runs a 60-second repeating tick; computes a target fraction of the total bot pool that should be online based on the current time window (`peak-hours.schedule` / `peak-hours.day-overrides`), then gradually sleeps (quietly removes) or wakes (respawns) AFK bots to match that fraction. Stagger spread is configured via `peak-hours.stagger-seconds`. Requires `swap.enabled: true`. Integrates with `BotSwapAI` — swapped-out bots count toward the pool so short swap absences don't distort the math. Sleeping-bot state is persisted crash-safely via `fpp_sleeping_bots` DB table (populated on every queue mutation; cleared on `wakeAll()`). Crash recovery: `restoreSleepingBotsFromDatabase(db)` repopulates the in-memory deque at startup. Hot-reload: `reload()` wakes all sleeping bots then restarts the tick if still enabled. Key public API: `start()`, `shutdown()`, `reload()`, `wakeAll()`, `wakeBotByName(name)`, `putBotToSleepByName(name)`, `forceCheck()`, `getSleepingCount()`, `getTotalPool()`, `getSleepingEntries()`, `getSleepingNames()`, `isSleeping(name)`, `getCurrentWindowLabel()`, `computeTargetFraction()`, `getSecondsToNextWindow()`, `getNextWindowFraction()`, `isRunning()`. Accessed via `plugin.getPeakHoursManager()`. Inner record: `PeakHoursManager.SleepingBot(name, loc)`.
- `fakeplayer/FakePlayer.java` — domain object for a single active bot (name, UUID, profile, spawnLocation, displayName, skinName, frozen state). `getPhysicsEntity()` and `getPlayer()` both return the single NMS ServerPlayer entity. Additional fields: `luckpermsGroup` (per-bot LP group override, `null` = not yet resolved; get/set via `getLuckpermsGroup()`/`setLuckpermsGroup()`), `rawDisplayName` (the `{bot_name}` slot before LP prefix/suffix are added; used as the stable base for `refreshLpDisplayName` rebuilds), `bodyless` (when `true` the bot exists in tab-list/chat only, no entity in world — used for console spawns without location data), `botType` (`BotType` enum — `AFK` = default passive bot; `PVP` = PvP bot (name prefixed `pvp_<name>`, currently restricted to internal dev UUID); get/set via `getBotType()`/`setBotType()`), `chatEnabled` (per-bot chat mute flag, default `true`; `false` = permanently silenced; get/set via `isChatEnabled()`/`setChatEnabled()`), `chatTier` (per-bot chat activity tier override, `null` = random; values `"quiet"`, `"passive"`, `"normal"`, `"active"`, `"chatty"`; get/set via `getChatTier()`/`setChatTier()`), `aiPersonality` (persistent AI personality name from `personalities/` folder; `null` = use config default; get/set via `getAiPersonality()`/`setAiPersonality()`), `rightClickCommand` (command executed when player right-clicks bot body; `null` = open inventory GUI), `headAiEnabled` (per-bot head-tracking toggle), `pickUpItemsEnabled` / `pickUpXpEnabled` (per-bot item/XP pickup toggles, initialized from `Config.bodyPickUpItems()` / `Config.bodyPickUpXp()`). Note: `botIndex` has been removed; `nametagEntity` remains as a legacy-compat field (no-op for NMS players).
- `fakeplayer/FakePlayerBody.java` — NMS ServerPlayer spawning/removal. Entities tagged via PDC keys for identification.
- `fakeplayer/NmsPlayerSpawner.java` — reflection-based NMS `ServerPlayer` factory (cached class/method/constructor lookups); bypasses Mojang auth by creating the player directly in JVM memory.
- `fakeplayer/NmsReflection.java` — shared NMS reflection bootstrap (CraftBukkit/NMS class resolution); initialised once, thread-safe.
- `fakeplayer/PathfindingService.java` — centralised shared navigation service used by all nav-capable commands. `Owner` enum (`MOVE`, `MINE`, `PLACE`, `USE`, `SYSTEM`). `NavigationRequest` record fields: `owner`, `destinationSupplier`, `arrivalDistance`, `recalcDistance`, `maxNullPathRecalculations`, `onArrive`, `onCancel`, `onPathFailure`, `lockOnArrival` (optional `Location` — when non-null, `cleanup()` atomically applies `manager.lockForAction()` before `onArrive` fires, eliminating the one-tick gap where neither the nav-lock nor the action-lock is held). Backward-compatible 8-arg constructor sets `lockOnArrival = null`. Public API: `navigate(fp, request)`, `cancel(uuid)`, `cancelAll()`, `cancelAll(owner)`, `isNavigating(uuid)`, `isNavigating(uuid, owner)`, `getOwner(uuid)`. Exposed via `plugin.getPathfindingService()`. Constructor: `new PathfindingService(plugin, fakePlayerManager)`.
- `gui/BotSettingGui.java` — per-bot 6-row chest GUI opened by shift-right-clicking a bot entity (when `Config.isBotShiftRightClickSettingsEnabled()`). Categories: ⚙ General (frozen toggle, look-at-player toggle, rename action), 💬 Chat (chat enabled, tier, AI personality), ⚔ PvP, 📋 Cmds (set/clear RC command), ⚠ Danger (delete bot). Uses `BotRenameHelper` for the rename action. Instantiated in `onEnable`, passed to `InventoryCommand` constructor, registered as listener.
- `command/PersonalityCommand.java` — `/fpp personality [list|reload|<bot> <set <name>|reset|show>]` (alias `persona`). Assigns AI personality files from `personalities/` to bots; persisted to DB via `updateBotAiPersonality()`. Permission: `Perm.PERSONALITY = "fpp.personality"`.
- `command/PlaceCommand.java` — `/fpp place <bot> [once|stop]  |  stop`. Classic continuous-place mode (like `/fpp mine` but placing): bot stands locked at the sender's position and places blocks it's looking at. Supports `once` for single-placement. **Persistence API** (used by `BotPersistence`): `getActivePlaceLocation(UUID)` → current dest+yaw+pitch lock location (null = not placing), `isActivePlaceOnce(UUID)` → once flag, `resumePlacing(FakePlayer, boolean once, Location)` → restores classic place mode after restart (navigates to dest first if needed). Area-fill mode (`AREA_MODE_ENABLED = false`) is disabled. Area subcommands (`--pos1`, `--pos2`, `--block`, `start`) are blocked while disabled. Permission: `Perm.PLACE = "fpp.place"`.
- `command/StorageCommand.java` — `/fpp storage <bot> [name|--list|--remove <name>|--clear]`. Registers named supply container locations used by `MineCommand` and `PlaceCommand` for restocking. Persistent via `StorageStore` (`data/storage-points.yml`). Permission: `Perm.STORAGE = "fpp.storage"`.
- `command/StorageStore.java` — persistence helper for per-bot `StoragePoint` (name + location) records; `getStorages(botName)`, `addStorage()`, `removeStorage()`, `clear()`. Saved to `data/storage-points.yml`; loaded in `onEnable` before command registration via `storageStore.load()`.
- `command/MineSelectionStore.java` — persists mine area selections (pos1/pos2 + active flag) to `data/mine-selections.yml`; selections survive restarts so `/fpp mine <bot> --start` works after a restart without re-selecting corners. Loaded in `onEnable` before command registration via `mineSelectionStore.load()`.
- `fakeplayer/RemoteBotCache.java` — thread-safe registry of bots physically running on other proxy servers. Always initialised (even in LOCAL mode). Populated from DB at startup (NETWORK mode) and updated via `BOT_SPAWN`/`BOT_DESPAWN` plugin messages. Exposed via `FakePlayerPlugin.getRemoteBotCache()`.
- `fakeplayer/RemoteBotEntry.java` — immutable record (`serverId`, `uuid`, `name`, `displayName`, `packetProfileName`, `skinValue`, `skinSignature`) for a remote bot; `hasSkin()` checks whether texture data is present.
- `fakeplayer/SkinProfile.java` — immutable snapshot of a resolved Mojang skin (base64 value + signature + source label).
- `config/BotNameConfig.java`, `config/BotMessageConfig.java` — bot name & message pools which are initialised in `onEnable`.
- `fakeplayer/SkinRepository.java` / `fakeplayer/SkinFetcher.java` — skin-loading subsystem (lazy singleton; explicitly initialised via `SkinRepository.get().init(this)` in `onEnable` after BotMessageConfig but before database init; must be ready before any bot spawn including persistence restore).
- `fakeplayer/network/FakeConnection.java` — NMS `Connection` subclass used for every bot. Created before `placeNewPlayer()` via `new FakeConnection(loopbackAddress)`; sets `this.channel = new FakeChannel(...)` and calls `Connection.configureSerialization()` on the no-op pipeline. Overrides all three `send()` overloads as no-ops to eliminate vanilla connection side-effects (protocol-state registers, pending-ack queues, callback notifications) that caused a phantom "Anonymous User" entry (UUID `0000…0000`) in the player list. `isConnected()` always returns `true` so NMS never treats the bot as disconnected.
- `fakeplayer/network/FakeServerGamePacketListenerImpl.java` — extends NMS `ServerGamePacketListenerImpl`; `send()` is a no-op that discards all outbound packets. Exceptions: `ClientboundSetEntityMotionPacket` (knockback) is intercepted and applied server-side via reflected `getMovement()`/`lerpMotion()` (1.21.9+) or `getXa()/Ya()/Za()` fallback. Injected into `ServerPlayer.connection` immediately after `placeNewPlayer()` so `awaitingPositionFromClient` stays `null` — preventing the server from snapping the bot back to its spawn position.
- `fakeplayer/SkinProfileInjector.java` — reflection helper that injects a `textures` property into an authlib `GameProfile`; used by both NMS body spawns and tab-list packet profiles for consistent skin application. Call `SkinProfileInjector.apply(gameProfile, skinProfile)`.
- `fakeplayer/BotIdentityCache.java` — persistent registry mapping `(bot_name, server_id) → stable UUID`. Ensures a bot with the same name always rejoins with the same UUID (LuckPerms data, inventory, session history stay linked). Storage priority: (1) in-memory `ConcurrentHashMap` (always checked first), (2) `fpp_bot_identities` DB table when DB is enabled, (3) `data/bot-identities.yml` YAML fallback when DB is disabled. Initialized in `onEnable` after the DB, set on `FakePlayerManager` via `setIdentityCache()`; `FakePlayerManager.resolveUuid(name)` delegates here for every fresh spawn. `prime(name, uuid)` pre-populates the cache from `spawnRestored()` so restart-restored bots don't hit the DB a second time. **Identity scope:** per-server — cross-server UUID sharing is intentionally not supported. Keys are lowercased for case-insensitive lookups.
- `gui/SettingGui.java` — interactive 3-row chest GUI (`/fpp settings`). 7 categories: General, Body, Chat, Swap, Peak Hours, PvP, Pathfinding; up to 18 settings per page. Toggle items flip their boolean in-place on click; numeric/double items close the chest and prompt the player to type a value in chat (message cancelled — invisible to other players, 60-second timeout auto-cancels). After each change, calls `plugin.saveConfig()`, `Config.reload()`, and `applyLiveEffect(configKey)` — which mirrors `ReloadCommand`'s subsystem calls for instant live effect without a full `/fpp reload`. **Reset button** (slot 18): resets all settings on the current page to JAR defaults. Per-player session state stored in `sessions` map (keyed by UUID). Chat-input sessions tracked separately in `chatSessions`; `pendingChatInput` / `pendingRebuild` sets prevent false `InventoryCloseEvent` handling during transitions. `SettingGui` is a singleton instantiated in `onEnable`, passed to `SettingCommand`, and registered as a listener via `registerEvents(settingGui, this)`. **Permission:** `Perm.SETTINGS = "fpp.settings"` (default `op`).
- `lang/Lang.java` — message rendering. Loads `language/en.yml`, supports named and positional placeholders. Use `Lang.get("key", "name", "value")` for an Adventure `Component` or `Lang.raw("key", ...)` for a raw string. Hot-reloadable via `/fpp reload`.
- `permission/Perm.java` — central permission registry. All permission nodes declared as constants (e.g. `Perm.SPAWN`, `Perm.DELETE`, `Perm.USER_SPAWN`). Use `Perm.has()`, `Perm.hasOrOp()`, `Perm.resolveUserBotLimit()` helpers. Never hard-code permission strings in commands.
- `database/DatabaseManager.java`, `database/BotRecord.java` — session history and analytics. Supports SQLite (default, WAL mode) and MySQL. Writes are serialised on a background thread; positions are batch-flushed on a configurable interval. Controlled by `Config.databaseEnabled()` and `Config.mysqlEnabled()`. **Schema version 13.** Tables: `fpp_bot_sessions` (session history), `fpp_active_bots` (restart persistence source of truth — v8→v9 added `frozen`, `chat_enabled`, `chat_tier`, `right_click_cmd`; v9→v10 added `ai_personality VARCHAR(64)`; v10→v11 added `pickup_items BOOLEAN`, `pickup_xp BOOLEAN`), `fpp_sleeping_bots` (peak-hours crash-safe sleeping-bot queue — `sleep_order`, `bot_name`, `world_name`, `pos_x/y/z`, `pos_yaw/pitch`, `server_id`; populated whenever the peak-hours sleep queue changes, cleared on `wakeAll()`), `fpp_bot_identities` (stable name→UUID registry — `bot_name`, `server_id`, `bot_uuid`, `created_at`; PRIMARY KEY `(bot_name, server_id)`; backfilled from earliest session history on schema upgrade), **`fpp_bot_tasks`** (v12→v13 — active task state for mine/use/place/patrol per bot; one row per task type per bot; PRIMARY KEY `(bot_uuid, server_id, task_type)`; saved on shutdown via `saveBotTasks()`, loaded on startup via `loadBotTasksForThisServer()`, cleared after loading via `clearBotTasks()` — same double-restore prevention pattern as `clearActiveBots()`), `fpp_meta` (schema version). Task API: `saveBotTasks(List<BotTaskRow>)`, `loadBotTasksForThisServer()` (synchronous), `clearBotTasks()`. `BotTaskRow` record: `botUuid`, `serverId`, `taskType` (`MINE`/`USE`/`PLACE`/`PATROL`), `worldName`, `posX/Y/Z`, `posYaw/Pitch`, `onceFlag`, `extraStr` (patrol route name), `extraBool` (patrol random). Other API: `saveSleepingBots(List<SleepingBotRow>)`, `loadSleepingBots()`, `clearSleepingBots()`, `lookupBotUuid(botName, serverId)` (synchronous read → `UUID` or `null`), `registerBotUuid(botName, uuid, serverId)` (enqueued write — INSERT OR IGNORE so first UUID always wins), `updateBotAiPersonality(String botUuid, String personality)` (enqueued write).
- `util/TabListManager.java`, `util/UpdateChecker.java` — tab header/footer management and async update checker initialized during startup.

- Migration runs before config init: `ConfigMigrator.migrateIfNeeded(this)` in `onEnable` (see `FakePlayerPlugin.java`).
- Config usage: call `Config.init(this)` then use accessors like `Config.fakeChatEnabled()` in code. Tab-list visibility accessor: `Config.tabListEnabled()` (key `tab-list.enabled`, default `true`) — when `false`, bots are never added to any player's tab list; the 20-tick display-name refresh loop, `syncToPlayer`, and the respawn re-add in `FakePlayerEntityListener` all check this flag before sending any `sendTabListAdd` packets. Toggling via `/fpp reload` calls `FakePlayerManager.applyTabListConfig()` which immediately removes or re-adds all active bots' tab entries for all online players. **Key accessors:** `Config.fakeChatEnabled()` (`fake-chat.enabled`, default `false`); `Config.fakeChatRequirePlayer()` (`fake-chat.require-player-online`, default `true`); `Config.fakeChatMessages()` (messages from `bot-messages.yml`). Local bots with a body use `player.chat(message)` so the server's real chat pipeline handles formatting. Bodyless bots and remote proxy bots use `Config.fakeChatRemoteFormat()` (`fake-chat.remote-format`, default `"<yellow>{name}<dark_gray>: <white>{message}"`) — supports `{name}` and `{message}` placeholders. `Config.bodyPushable()` (`body.pushable`, default `true`); `Config.bodyDamageable()` (`body.damageable`, default `true`); `Config.bodyPickUpItems()` (`body.pick-up-items`, default `false` — bots do not pick up items); `Config.bodyPickUpXp()` (`body.pick-up-xp`, default `true` — bots pick up XP orbs; set `false` to prevent all orb pickup; also honoured by the `XpCommand` post-collection cooldown via `BotXpPickupListener`); `Config.warningsNotifyAdmins()` (`messages.notify-admins-on-join`, default `true`) — send compatibility warning to online admins on startup. **Swap system config (v1.5.17):** `Config.swapMinOnline()` (`swap.min-online`, default `0` — minimum bots that must stay online; swap skips if removing one would drop below this floor); `Config.swapRetryRejoin()` (`swap.retry-rejoin`, default `true` — auto-retry failed rejoins); `Config.swapRetryDelay()` (`swap.retry-delay`, default `60` seconds — delay before retry). **Granular debug flags:** `Config.debugStartup()`, `Config.debugNms()`, `Config.debugPackets()`, `Config.debugLuckPerms()`, `Config.debugNetwork()`, `Config.debugConfigSync()`, `Config.debugSkin()`, `Config.debugDatabase()`, `Config.debugChat()`, `Config.debugSwap()` — each maps to `logging.debug.<category>` in config (also triggered by global `debug: true`). **Network/proxy accessors:** `Config.databaseMode()` (`database.mode`, `"LOCAL"` or `"NETWORK"`); `Config.isNetworkMode()` — shorthand for `databaseEnabled() && databaseMode()=="NETWORK"`; `Config.serverId()` (`database.server-id`, falls back to legacy `server.id`); `Config.configSyncMode()` (`config-sync.mode`, one of `DISABLED` / `MANUAL` / `AUTO_PULL` / `AUTO_PUSH`). Server-specific keys (`database.server-id`, `database.mysql.*`, `debug`) are never pushed/pulled by `ConfigSyncManager`. **Performance config (v1.5.17):** `Config.positionSyncDistance()` (`performance.position-sync-distance`, default `128.0` — max distance for position sync packets; `0` = unlimited). Note: `Config.tabListNameFormat()` and the `bot-name.tab-list-format` config key were **removed** in config v38 — LP manages prefix/suffix natively for NMS ServerPlayer entities and the display name packet carries only the bot name itself.
- Language messages: always use `Lang.get("key")` (returns Adventure `Component`) or `Lang.raw("key")` (returns `String`) — never hard-code user-facing text. Named placeholders: `Lang.get("freeze-frozen", "name", fp.getDisplayName())`. All keys live in `src/main/resources/language/en.yml`.
- Permission checks: use `Perm.has(sender, Perm.SPAWN)` or `Perm.hasOrOp(sender, Perm.OP)` — never hard-code node strings. **All permissions declared in `Perm.java` as constants AND in `plugin.yml` for LuckPerms auto-discovery.** Per-user bot limits resolved via `Perm.resolveUserBotLimit(sender)` (scans `fpp.spawn.limit.1` … `fpp.spawn.limit.100`; returns -1 if none set, caller falls back to `Config.userBotLimit()`). **Permission structure:** `fpp.op` grants all commands (admin wildcard, default: op); `fpp.use` grants user-tier commands: spawn (with 1-bot limit), tph, xp, info-user (default: true — all players). Bot limits assigned via `fpp.spawn.limit.<num>` nodes (1-100); `fpp.spawn.limit.1` is included in `fpp.use` by default. New command permission constants: `Perm.INVENTORY = "fpp.inventory"`, `Perm.MOVE = "fpp.move"`, `Perm.CMD = "fpp.cmd"`, `Perm.MINE = "fpp.mine"`, `Perm.STORAGE = "fpp.storage"`, `Perm.PLACE = "fpp.place"`, `Perm.PERSONALITY = "fpp.personality"`, `Perm.USE_CMD = "fpp.useitem"`, `Perm.WAYPOINT = "fpp.waypoint"`, `Perm.USER_XP = "fpp.xp"` (user-tier, included in `fpp.use`), `Perm.USER_INFO = "fpp.info.user"`, `Perm.USER_TPH = "fpp.tph"`, `Perm.MIGRATE = "fpp.migrate"`. When adding new commands: (1) add constant to `Perm.java`, (2) add full entry to `plugin.yml` with description and default, (3) add to `fpp.op` children list in `plugin.yml`, (4) use constant in command's `getPermission()` and `canUse()` methods. **LuckPerms tab-completion depends on `plugin.yml` declarations** — missing entries won't appear in `/lp permission set fpp.<tab>`.
- Command registration: plugin registers commands via `commandManager.register(new SpawnCommand(...))`; `HelpCommand` is auto-registered inside `CommandManager`, and `/fpp help` reads live from the registered command list, so new sub-commands need no separate help-menu wiring. The plugin then calls `getCommand("fpp").setExecutor(commandManager)` (see `onEnable`).
- Listener registration: `getServer().getPluginManager().registerEvents(new PlayerJoinListener(this, fakePlayerManager), this)` — listeners receive plugin + manager references.
- Persistence lifecycle: restore on enable via `botPersistence.purgeOrphanedBodiesAndRestore(fakePlayerManager)` (purges orphaned entities then calls `restore()` 5 ticks later) and save on disable via `botPersistence.save(fakePlayerManager.getActivePlayers())`. `BotPersistence.saveAsync()` also available for non-shutdown saves (snapshots locations on main thread, writes async). When DB is enabled, the `fpp_active_bots` table is the **primary** restore source (`DatabaseManager.getActiveBotsForThisServer()`); `data/active-bots.yml` is the fallback for no-DB or empty-table cases. **Task persistence (v1.6.2+):** active tasks (mine/use/place/patrol) are saved to `fpp_bot_tasks` (DB primary) and `data/bot-tasks.yml` (YAML fallback) on shutdown via `writeTaskSnapshot()`; loaded back in `loadTasksFile()` which tries DB first, falls back to YAML. `clearBotTasks()` is called immediately after loading so tasks don't re-restore on the next restart. `BotPersistence` injections (must happen AFTER all commands created, BEFORE `purgeOrphanedBodiesAndRestore`): `setMoveCommand`, `setMineCommand`, `setPlaceCommand`, `setUseCommand`, `setWaypointStore`.
- **Performance optimization (v1.5.17):** `FakePlayerManager.getByName()` is now O(1) via secondary `nameIndex` map (`ConcurrentHashMap<String, FakePlayer>`, lowercased keys) — was O(n) linear scan. The index is maintained at all add/remove sites. Additionally, position sync packets are distance-culled via `Config.positionSyncDistance()` (`performance.position-sync-distance`, default `128.0` blocks, `0` = unlimited) — only players within range receive per-tick position updates, reducing packet overhead on large servers.
- Metrics & soft-deps: metrics are initialized with `FppMetrics` when `Config.metricsEnabled()`; PlaceholderAPI expansion registered conditionally in `onEnable`.
- Compatibility checks: `CompatibilityChecker.check()` (in `util/CompatibilityChecker.java`) **always returns a non-restricted result** — the old Paper-detection and minimum MC version gate has been removed. `check()` is **not called in `onEnable`**; `compatibilityRestricted` and `compatibilityWarningMessage` fields have been removed from `FakePlayerPlugin`. The class retains two general-purpose utilities: `CompatibilityChecker.extractMcVersion()` (returns detected MC version string) and `CompatibilityChecker.isVersionAtLeast(version, required)` (component-wise numeric comparison). `buildWarningComponent(result)` always returns `null`. **Version gate (separate from `check()`):** `FakePlayerPlugin` directly calls `CompatibilityChecker.isVersionAtLeast(detectedMcVersion, "1.21.12")` in `onEnable` to set `versionUnsupported = true` when the server runs MC 1.21.12+. New fields: `versionUnsupported` (boolean) and `detectedMcVersion` (String); new accessors: `plugin.isVersionUnsupported()` and `plugin.getDetectedMcVersion()`. When `versionUnsupported = true`: `CommandManager` blocks all sub-commands (bare `/fpp` still shows plugin info), `PlayerJoinListener` sends the `"version-unsupported-admin"` Lang warning to operators on every join, and a prominent banner is printed to the console at startup. FPP is tested up to MC 1.21.11; any newer version triggers this gate.
- Bot pools: `BotNameConfig.init(this)` and `BotMessageConfig.init(this)` are called during startup — the plugin loads `bot-names.yml` and `bot-messages.yml` early.
- Metrics / FastStats packaging: `FppMetrics` is initialized before the startup banner to reflect metrics status. FastStats jars are bundled under `src/main/resources/faststats/` (the POM treats them as binary resources, not shaded). Note: building requires JDK 21 (see `pom.xml` <java.version>21).
- Soft-dependencies declared in `plugin.yml`: `PlaceholderAPI`, `LuckPerms`, `WorldGuard` (all optional, auto-detected). Current `pom.xml` does not declare PacketEvents.
- Entity stack: each bot is a single NMS ServerPlayer entity (created via `NmsPlayerSpawner`) with full player functionality including chat, tab-list presence, and realistic movement. Use PDC keys for identification during event handling.
- Database gate: check `Config.databaseEnabled()` before any `DatabaseManager` call; `databaseManager` may be `null` at runtime (disabled or failed to init). `databaseManager.recordAllShutdown()` must be called in `onDisable` before `databaseManager.close()`.
- Text formatting: `TextUtil.colorize(string)` parses MiniMessage tags and legacy `&`/`§` codes into an Adventure `Component`. `TextUtil.format(string)` additionally applies small-caps Unicode conversion. **Supports all modern color formats:** MiniMessage tags (`<rainbow>`, `<gradient:#FF0000:#0000FF>`), hex colors (`<#9782ff>`), LuckPerms gradient shorthand (`{#fffff>}text{#00000<}`), and mixed formats (`&7[<#9782ff>Phantom</#9782ff>&7]`).
- LuckPerms integration: `LuckPermsHelper` reads resolved prefix/suffix directly from LP's in-memory `CachedMetaData` — bots are real NMS `ServerPlayer` entities, LP handles them natively. Core helpers: `LuckPermsHelper.isAvailable()`, `LuckPermsHelper.groupExists(groupName)`, `LuckPermsHelper.getAllGroupNames()`, `LuckPermsHelper.buildGroupSummary()`. Runtime group assignment: `LuckPermsHelper.setPlayerGroup(uuid, groupName)` (works online or offline) or `LuckPermsHelper.applyGroupToOnlineUser(uuid, groupName)` (prefers the live `User` instance so `UserDataRecalculateEvent` fires immediately — preferred when bot is already spawned). Prefix/suffix reads: `LuckPermsHelper.getResolvedPrefix(uuid)` / `LuckPermsHelper.getResolvedSuffix(uuid)` (synchronous, requires bot to be online). Async group read from storage: `LuckPermsHelper.getStoredPrimaryGroup(uuid)`. Online-only read: `LuckPermsHelper.getPrimaryGroup(uuid)`. Force LP metadata refresh: `LuckPermsHelper.refreshUserCache(uuid)`. `LuckPermsHelper.invalidateCache()` is `@Deprecated` (no-op — LP handles caching natively). **Event subscription:** `LuckPermsHelper.subscribeLpEvents(plugin, manager)` subscribes to `UserDataRecalculateEvent` so bot display names auto-update whenever a group changes (LP command, API, etc.); called directly in `onEnable` (not deferred). `LuckPermsHelper.unsubscribeLpEvents()` is called in `onDisable` to prevent memory leaks. **LP group assignment:** `Config.luckpermsDefaultGroup()` (`luckperms.default-group`, default `""`) — optional LP group assigned at spawn; blank = `"default"`. Use `/fpp lpinfo [bot-name]` for in-game diagnostics. Check LP availability via `LuckPermsHelper.isAvailable()` before any API call. **LP injection flow:** `finishSpawn` calls `LuckPermsHelper.ensureGroupBeforeSpawn(botUuid, cfgGroup)` **async before the NMS body spawns** — writes the group to LP storage so `PlayerJoinEvent` listeners (TAB plugin, etc.) see the correct group immediately. If the bot already has a non-default group in storage (prior `/fpp rank`), that group is preserved unless `luckperms.default-group` overrides it. Once LP completes, `spawnBodyAndFinish(fp, spawnLoc)` runs on the main thread. After the body spawns, `LuckPermsHelper.applyGroupToOnlineUser(uuid, group)` is called 5 ticks later; after that completes, `refreshLpDisplayName(fp)` is called 2 ticks later to update the tab-list display name. `FakePlayer.rawDisplayName` is the stable `{bot_name}` base for rebuilds. Call `FakePlayerManager.refreshLpDisplayName(fp)` to manually rebuild display name + resend tab-list update packets. **Scoreboard team override:** All bots are placed in the `~fpp` scoreboard team (managed by `BotTabTeam`) which sorts AFTER all other teams, ensuring bots always appear below real players regardless of LP group weight. Note: `LpData` record, `getBotLpData()`, `buildPacketProfileName()`, `invalidateGroup()`, `refreshRankList()`, and the packet-prefix-char weight-rank ordering system have all been removed. **ClassLoader guard:** `FakePlayerPlugin` caches a `luckPermsAvailable` boolean (set once in `onEnable()` via `Bukkit.getPluginManager().getPlugin("LuckPerms") != null`) accessible via `plugin.isLuckPermsAvailable()`. **Never reference `LuckPermsHelper` or any LP API class unless this returns `true`** — Paper's `PluginClassLoader` eagerly resolves LP API class references during bytecode verification, so even calling `LuckPermsHelper.isAvailable()` on a server without LP installed throws `NoClassDefFoundError`. The pattern everywhere LP is needed: `if (plugin.isLuckPermsAvailable()) { LuckPermsHelper.doSomething(...); }`.

Key methods: `WorldGuardHelper.isPvpAllowed(location)` — queries active WG region flags; returns `true` when no region explicitly sets PVP to DENY (fail-open: unprotected areas always allow PvP, only explicit DENY blocks damage); `WorldGuardHelper.findSafeLocation(world)` — returns the world spawn if PvP is allowed there, otherwise `null`. Used by `FakePlayerEntityListener` at `LOWEST` priority to cancel player-sourced PvP damage to bots inside no-PvP regions. **ClassLoader guard:** `FakePlayerPlugin` caches `worldGuardAvailable` (set once via `Bukkit.getPluginManager().getPlugin("WorldGuard") != null`) accessible via `plugin.isWorldGuardAvailable()`. **Never reference `WorldGuardHelper` unless this returns `true`** — same NoClassDefFoundError risk as LuckPerms. - Listener registration: `getServer().getPluginManager().registerEvents(new PlayerJoinListener(this, fakePlayerManager), this)` — listeners receive plugin + manager references. `FakePlayerKickListener` (guards all bots from automated kicks like anti-idle/anti-AFK) is always registered; all other listeners (`FakePlayerEntityListener`, `BotCollisionListener`, etc.) are also always registered — the old `compatibilityRestricted` skip is gone.
- `onDisable` sequence: `chunkLoader.releaseAll()` → `botChatAI.cancelAll()` → `botConversationManager.clearAll()` → **`peakHoursManager.shutdown()`** (wakes all sleeping bots so they appear in `activePlayers`) → `botSwapAI.cancelAll()` → **save persistence** (sleeping bots are now active and captured) → `LuckPermsHelper.unsubscribeLpEvents()` → `fakePlayerManager.removeAllSync()` (also calls `pvpAI.cancelAll()` internally) → `tabListManager.shutdown()` → `botTabTeam.destroy()` → `databaseManager.recordAllShutdown()` + `close()` → `fppMetrics.shutdown()` → unregister VelocityChannel messaging channels. **Critical ordering:** `peakHoursManager.shutdown()` MUST run before `botPersistence.save()` so sleeping bots are woken and included in the persistence file.
- Update notifications to late-joining admins: stored as a `Component` on `FakePlayerPlugin.getUpdateNotification()` / `setUpdateNotification()`; `PlayerJoinListener` delivers it via `Perm.hasOrOp(player, Perm.OP)`.

Returns a `SyncResult` record (`fileName`, `keysAdded`, `hasChanges()`, `count()`). Called automatically inside `Lang.reload()`, `BotNameConfig.reload()`, and `BotMessageConfig.reload()` on every startup and `/fpp reload`. Force-run from the command line with `/fpp migrate lang`, `/fpp migrate names`, or `/fpp migrate messages`. When adding new keys to `en.yml`, `bot-names.yml`, or `bot-messages.yml`, they will be automatically synced to existing user files on next reload — no manual migration step needed.

Helpers: `manager.isOnCooldown(uuid)`, `manager.getRemainingCooldown(uuid)` (seconds), `manager.recordSpawnCooldown(uuid)`.

Both methods prune to the most recent `MAX_BACKUPS` (10) automatically.
- Additional `Perm` helpers: `Perm.missing(sender, node)` returns `true` when the sender lacks the node (negation of `has()`); `Perm.hasAny(sender, node1, node2, ...)` returns `true` when the sender has at least one of the listed nodes.
- Physical-body API: `FakePlayerManager.physicalBodiesEnabled()` — returns `Config.spawnBody()`. (The old `&& !plugin.isCompatibilityRestricted()` check has been removed.) Use this instead of checking `Config.spawnBody()` inline when deciding whether NMS ServerPlayer bodies are active.
- Display-name pipeline: `FakePlayerManager.finalizeDisplayName(rawName, botName)` — private helper that expands PAPI `%placeholders%` via `PlaceholderAPI.setPlaceholders(null, ...)` (null player = server-wide), then calls `sanitizeDisplayName()`. **Note:** the `{prefix}/{bot_name}/{suffix}` tab-list format substitution (previously driven by `bot-name.tab-list-format`) was removed in config v38 — LP manages prefix/suffix natively for real NMS ServerPlayer entities; the display name packet carries only the bot name. Called at every display-name build site: `spawnUserBot`, `spawn`, `spawnRestored`, `validateUserBotNames`. Public entry point for post-LP-assignment rebuilds: `FakePlayerManager.refreshLpDisplayName(fp)` (also has a private retry variant `refreshLpDisplayNameWithRetry` for LP metadata race conditions).
- Body damageable/pushable live-reload: `FakePlayerManager.applyBodyConfig()` — called on `/fpp reload`; handles damage/pushable configuration for all active NMS ServerPlayer entities. `BotCollisionListener` also guards all three push paths (`onEntityDamageByEntity`, `onPlayerMove`, `tickBotSeparation`) with `if (!Config.bodyPushable()) return;`.
- Swim AI: `Config.swimAiEnabled()` (`swim-ai.enabled`, default `true`). Each tick the main loop calls `NmsPlayerSpawner.setJumping(bot, bot.isInWater() || bot.isInLava())` so bots automatically swim upward when submerged, mimicking a player holding spacebar. Set `swim-ai.enabled: false` to let bots sink/drown instead.

4) Common tasks & minimal steps (how to implement typical changes)
- Add a new command:
  - Create `src/main/java/me/bill/fakePlayerPlugin/command/MyCommand.java` implementing the command interface used by `CommandManager`.
  - Register it in `FakePlayerPlugin.onEnable()` with `commandManager.register(new MyCommand(...))`.
  - No manual help update is needed — `HelpCommand` reads live from `CommandManager`.
  - Ensure `plugin.yml` contains the `fpp` command or sub-command docs (command parsing done by `CommandManager`).
- Add a config option:
  - Add default to `src/main/resources/config.yml`.
  - If needed, update `Config` accessor and `ConfigMigrator` to preserve old configs.
  - New options should be validated in `util/ConfigValidator.java` if they can be misconfigured.
- Add a language message:
  - Add the key and default text to `src/main/resources/language/en.yml`.
  - Access it in code with `Lang.get("your-key")` or `Lang.raw("your-key", "placeholder", "value")`.
  - The file is hot-reloadable — `/fpp reload` picks up changes without restart.
- Add a permission node:
  - Declare a new `public static final String` constant in `permission/Perm.java` (e.g. `public static final String MY_FEATURE = "fpp.myfeature";`).
  - Add a full entry under `permissions:` in `plugin.yml` with description and default value (e.g. `op`, `true`, or `false`).
  - Add the node to the `fpp.op` wildcard's `children:` list in `plugin.yml` (so admins inherit it automatically).
  - Use the constant in your command's `getPermission()` and `canUse()` methods: `Perm.has(sender, Perm.MY_FEATURE)`.
  - **Critical:** LuckPerms reads `plugin.yml` at startup for tab-completion — permissions missing from `plugin.yml` won't appear in `/lp permission set fpp.<tab>`.
  - Test: restart server, run `/lp user <name> permission set fpp.` and verify your new permission appears in tab-completion.

    - If changing skin behaviour or adding skins: ensure `SkinRepository`/`SkinFetcher` are respected — add skin files to `src/main/resources/skins/` for defaults or `plugins/FakePlayerPlugin/skins/` at runtime and call `/fpp reload` to refresh. The startup code also drops a `skins/README.txt` inside the plugin data folder on first run. **Note (v1.5.4):** `skin.guaranteed-skin` default changed from `true` → `false`; bots with non-Mojang names now use Steve/Alex skins by default. `skin.fallback-pool` (`Config.skinFallbackPool()`) and `skin.fallback-name` (`Config.skinFallbackName()`) still exist as auto-mode fallbacks for guaranteed-skin resolution.

```powershell
mvn -DskipTests clean package
```
The `maven-antrun-plugin` in `pom.xml` automatically copies `target/fpp-*-obfuscated.jar` to the deploy folder after ProGuard finishes. To deploy to a different path override the property:
```powershell
mvn -DskipTests clean package "-Ddeploy.dir=C:\path\to\server\plugins"
```
Note: requires JDK 21+ to compile (`pom.xml` `<java.version>21`). FastStats jars are bundled as binary resources in `src/main/resources/faststats/` (not shaded).

**IntelliJ build tool ("Build Artifacts"):** IntelliJ's native artifact builder bypasses Maven and ProGuard. Treat `target/fpp-*-obfuscated.jar` as the source artifact and use `pom.xml` as the source of truth for the current version; some repo docs/scripts still mention `1.4.22`. Workflow:
1. Run the **"Build Plugin (Obfuscated)"** Maven run configuration (or `mvn clean package`) — this runs ProGuard and auto-deploys
2. Optionally run **Build › Build Artifacts** to re-pack the already-obfuscated JAR to the output path without recompiling
- ⚠️ "Build Artifacts" alone (without Maven running first) will fail if `target/fpp-*-obfuscated.jar` does not exist

Double-click `BUILD.bat` in the project root as an alternative to Maven in terminal.

6) Debugging tips
- To reproduce startup ordering bugs, inspect `onEnable()` in `FakePlayerPlugin.java` — migration → config → Lang → **version gate** (`CompatibilityChecker.isVersionAtLeast(detectedMcVersion, "1.21.12")` — sets `versionUnsupported`; runs between Lang and BotNameConfig) → BotNameConfig/BotMessageConfig → ensureDataDirectories (`personalities/` folder also created here) → SkinRepository.init → database → **BotIdentityCache** → remote bot cache (NETWORK mode: pre-populate from DB) → ConfigSyncManager (NETWORK mode only) → FakePlayerManager+subsystems → commands (`pathfindingService` instantiated first; `WaypointStore.load()`, `StorageStore.load()`, `MineSelectionStore.load()` before command registration; **SettingGui instantiated here; `SettingCommand` registered; `settingGui` + `inventoryCommand` + `botSettingGui` registered as listeners immediately after**) → listeners → **HelpGui init + listener registration + `commandManager.setHelpGui(helpGui)`** → **BotChatAI + BotSwapAI + PeakHoursManager init** (PeakHoursManager started if `peak-hours.enabled && swap.enabled`) → **AI system init** (`AIProviderRegistry`, `BotConversationManager`, `PersonalityRepository.init()`) → `BotMessageListener` conditionally registered → VelocityChannel (plugin messaging) → periodic health check (6000 ticks) → ConfigValidator → TabListManager → BotTabTeam → PlaceholderAPI → LuckPerms (`subscribeLpEvents` — direct, not deferred) → **WorldGuard** (sets `worldGuardAvailable`) → UpdateChecker (purely async, never blocks startup) → Metrics → startup banner → persistence restore → **PeakHoursManager crash recovery** (`restoreSleepingBotsFromDatabase` — only when DB enabled and `persist-on-restart: true`). Note: `CompatibilityChecker.check()` is no longer called at startup — all features are enabled unconditionally. **`BotPersistence` injections** (must happen AFTER all commands created, BEFORE `purgeOrphanedBodiesAndRestore`): `botPersistence.setMoveCommand(moveCommand)`, `setMineCommand(mineCommand)`, `setUseCommand(useCommand)`, `setWaypointStore(waypointStore)`.
- To debug config migrations, read `util/ConfigMigrator.java` and check backups under `plugins/FakePlayerPlugin/backups/` created by `BackupManager`.
- For packet-level issues (spawn / entity problems) inspect `fakeplayer/PacketHelper.java` and `fakeplayer/FakePlayerManager.java`.
- For database issues: SQLite file is at `plugins/FakePlayerPlugin/data/fpp.db`; run `/fpp migrate status` to check schema version (current: 13), YAML sync health, and backup count. Full `/fpp migrate` subcommands: `backup` (create manual backup), `backups` (list stored backups), `status` (config version + file sync health + DB stats), `config` (re-run config migration chain), `lang` / `names` / `messages` (force-sync the respective YAML file from JAR), `db merge [file]` / `db export` / `db tomysql`. Set `database.enabled: false` in config to disable all DB I/O.
- Tab list header/footer and update checker: `TabListManager.start()` and `UpdateChecker.check(this)` are run during `onEnable()` — useful when diagnosing missing tab headers or unexpected update-checker output.

7) Integration & soft-dependencies
- PlaceholderAPI: `util.FppPlaceholderExpansion` registered if plugin present. **Full placeholder list (29+ total)** — server-wide: `%fpp_count%` (bot count — local + remote in NETWORK mode), `%fpp_local_count%` (this server's bots only), `%fpp_network_count%` (remote servers' bots only), `%fpp_max%` (global cap, `∞` if 0), `%fpp_real%` (non-bot players online), `%fpp_total%` (real + bots, network-wide), `%fpp_online%` (alias for `%fpp_total%`), `%fpp_frozen%`, `%fpp_names%` (comma-joined display names — local + remote in NETWORK mode), `%fpp_network_names%` (remote-server bot names only), `%fpp_chat%`, `%fpp_swap%`, `%fpp_skin%`, `%fpp_body%`, `%fpp_pushable%`, `%fpp_damageable%`, `%fpp_tab%`, `%fpp_max_health%`, `%fpp_persistence%`, `%fpp_version%`.
