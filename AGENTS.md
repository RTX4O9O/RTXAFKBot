# AGENTS.md — Guide for AI coding agents

Purpose: help automated agents quickly understand, modify, and verify the FakePlayerPlugin codebase.

1) Quick repo map
- `src/main/java/me/bill/fakePlayerPlugin/` — main code and entry points
- `src/main/resources/` — default `config.yml`, `plugin.yml`, `bot-names.yml`, `bot-messages.yml`, `secrets.yml` (AI API keys template — copied to `plugins/FakePlayerPlugin/secrets.yml` on first run; managed by `SecretsConfig`). Bundled sample personality prompts: `personalities/friendly.txt`, `personalities/grumpy.txt`, `personalities/noob.txt` — extracted to `plugins/FakePlayerPlugin/personalities/` on first run.
- `src/main/java/me/bill/fakePlayerPlugin/api/` — public addon API. Stable addon-facing interfaces: `FppApi`, `FppBot`, `FppAddonCommand`, `FppBotTickHandler`; Bukkit events under `api/event/`: `FppBotSpawnEvent`, `FppBotDespawnEvent`, `FppBotDeathEvent`, `FppBotRenameEvent`, `FppBotChatEvent`; internal wrappers live in `api/impl/`. Addons should obtain the API via `FakePlayerPlugin.getFppApi()` and depend only on the interfaces/events, never on `FakePlayer` or `FakePlayerManager`.
- `src/main/resources/language/en.yml` — all user-facing messages (loaded by `Lang.java`)
- `pom.xml` — build + jar metadata (`target/fpp-*.jar` produced; Maven artifact version may carry an extra `.1` patch qualifier, e.g. `1.6.6.1`, that is not reflected in `plugin.yml`'s `version:` field — use `pom.xml` as the canonical version)
- `frontend/`, root `package.json`, and `vercel.json` — separate Node/Vercel status + update API sidecar used by the plugin’s update-check fallback; not part of the Maven plugin jar build.
- `frontend/wiki/` — primary and only feature docs (Commands.md, Configuration.md, Getting-Started.md, Database.md, Language.md, Permissions.md, Skin-System.md, Swap-System.md, Bot-Behaviour.md, Bot-Messages.md, Bot-Names.md, Fake-Chat.md, FAQ.md, Migration.md, Home.md, Proxy-Support.md, Config-Sync.md, Placeholders.md, Peak-Hours.md, Changelog.md). The root `wiki/` directory has been removed — `frontend/wiki/` is now the single canonical wiki location.
- `src/main/java/me/bill/fakePlayerPlugin/gui/` — GUI systems: `SettingGui` (in-game settings, `/fpp settings`), `HelpGui` (paginated command GUI used by `/fpp help`; console still uses text fallback), and `BotSettingGui` (per-bot 6-row chest GUI; opened by shift-right-clicking a bot entity when `Config.isBotShiftRightClickSettingsEnabled()` — categories: General (frozen, head-AI, swim-AI, chunk-load-radius, pickup-items, pickup-xp, rename), Chat, PvP (coming-soon overrides), Cmds, Danger; uses `BotRenameHelper` for in-GUI rename; now enforces a per-bot edit lock so only one player can edit a bot at a time, reusing `inv-busy`; instantiated in `onEnable`, registered as listener, passed to `InventoryCommand` constructor).
- `src/main/java/me/bill/fakePlayerPlugin/util/` — helper subsystems and utilities (e.g. `FppMetrics`, `FppLogger`, `FppPlaceholderExpansion`, `TabListManager`, `BackupManager`, `UpdateChecker`, `ConfigValidator`, `DataMigrator`, `TextUtil`, `ConfigMigrator`, `LuckPermsHelper`, `CompatibilityChecker`, `YamlFileSyncer`, `BotTabTeam`, `WorldGuardHelper`, `BadwordFilter`, `BotRenameHelper`, `NameTagHelper`, `FlagParser`, `AttributionManager`, `AttributionApiManager`, `AttributeCompat`). **`AttributeCompat`** — version-safe `Attribute.MAX_HEALTH` resolver. Paper 1.21.3+ uses shorthand `MAX_HEALTH`; older servers use `GENERIC_MAX_HEALTH`. Resolved once at class-load via reflection. Use `AttributeCompat.maxHealth()` instead of the raw enum literal wherever `Attribute.MAX_HEALTH` is needed. **`NameTagHelper`** — integration shim for the NameTag soft-dep; inner records `BotSkin` and `NickData`; `canRename()` helper; **ClassLoader guard:** never call unless `plugin.isNameTagAvailable()` returns `true` (cached in `onEnable` via `Bukkit.getPluginManager().getPlugin("NameTag") != null`). **`FlagParser`** — reusable command arg/flag parser with deprecation aliases, duplicate detection, and conflict detection. Pattern: `new FlagParser(args).deprecate(...).conflicts(...).parse()` — use this in new commands instead of ad-hoc arg scanning. `BotRenameHelper` centralises the validate → delete → respawn → restore lifecycle shared by `RenameCommand` and `BotSettingGui`. **`AttributionManager`** — lightweight attribution protection; validates original author credit in `plugin.yml` on startup via `AttributionManager.validate(plugin)` (called in `onEnable`). Encoded int-array storage prevents trivial text-search edits. Public API: `getOriginalAuthor()`, `getAttributionMessage()`, `getModrinthLink()`, `getGithubLink()`, `quickAuthorCheck()`, `quickMessageCheck()`, `isIntegrityValid()`, `formatAuthors(List<String>)`. Prefers API-verified data from `AttributionApiManager` when available, falls back to built-in encoded values. **`AttributionApiManager`** — async API-based attribution verification; fetches official data from `fakeplayerplugin.xyz/api/attribution` on startup (`init(plugin)` called in `onEnable` after `AttributionManager.validate`). Caches response to `data/attribution-cache.json`; falls back to local cache then built-in encoded data when API unreachable. Public API: `init(Plugin)`, `hasData()`, `isApiReachable()`, `getAuthor()`, `getMessage()`, `getModrinthLink()`, `getGithubLink()`. **`BadwordFilter`** — validates and sanitises bot names. Word sources (merged at load time): (1) remote global baseline (`badword-filter.global-list-url`, default CMU list — enabled by default), (2) `badword-filter.words` inline list in `config.yml`, (3) `plugins/FakePlayerPlugin/bad-words.yml` (local YAML + custom regex `patterns:` block). Detection passes applied in order: raw (case-insensitive substring), leet (after `0→o 1→i 3→e 4→a …` normalisation), aggressive (`auto-detection.enabled: true` — duplicate-char collapse), regex (auto-generated repeat/separator patterns + custom patterns). Whitelist: `badword-filter.whitelist` exact names always pass. Auto-rename: `badword-filter.auto-rename: true` substitutes a random clean name instead of hard-blocking. Public API: `BadwordFilter.reload(plugin)`, `isAllowed(name)`, `sanitize(name)`, `findBadword(name)`, `getBadwordCount()`, `getWhitelistCount()`. Called during startup (`BadwordFilter.reload(this)` after `Config.init`, before `Lang.init`), on every `/fpp reload`, and in `SpawnCommand`/`BotRenameHelper` for per-name validation.
- `src/main/java/me/bill/fakePlayerPlugin/listener/` — Bukkit event listeners (`PlayerJoinListener`, `FakePlayerEntityListener`, `FakePlayerKickListener`, `BotCollisionListener`, `PlayerWorldChangeListener`, `ServerListListener`, `BotCommandBlocker`, `BotSpawnProtectionListener`, `BotXpPickupListener`, `BotMessageListener`). Note: `LuckPermsUpdateListener` was removed — LP now detects NMS bots as real players automatically. `BotCommandBlocker` prevents bots from executing any commands. `BotSpawnProtectionListener` prevents lobby/spawn plugins from teleporting bots during their 5-tick spawn grace period. `BotXpPickupListener` gates XP orb pickup (`PlayerPickupExperienceEvent`) and programmatic XP grants (`PlayerExpChangeEvent`) for bots via `Config.bodyPickUpXp()` and the `XpCommand` post-collection cooldown. `BotMessageListener` intercepts `/msg`, `/tell`, `/whisper` (and aliases `/w`, `/m`, `/t`, `/pm`, `/dm`, etc.) directed at bots, cancels the event, and routes to `BotConversationManager`; registered only when `Config.aiConversationsEnabled() && aiProviderRegistry.isAvailable()`.
- `src/main/java/me/bill/fakePlayerPlugin/fakeplayer/` — bot domain objects and subsystems (e.g. `FakePlayer`, `FakePlayerBody`, `FakePlayerManager`, `BotPersistence`, `ChunkLoader`, `SkinRepository`, `SkinFetcher`, `SkinProfile`, `SkinProfileInjector`, `NmsHelper`, `NmsPlayerSpawner`, `NmsReflection`, `BotSwapAI`, `BotChatAI`, `BotBroadcast`, `PacketHelper`, `RemoteBotCache`, `RemoteBotEntry`, `BotType`, `BotCombatMode`, `BotPvpAI`, `PeakHoursManager`, `BotPathfinder`, `PathfindingService`, `BotNavUtil`, `StorageInteractionHelper`, `SwimAI`). **`PathfindingService`** centralises all navigation loops previously duplicated across `MoveCommand`, `MineCommand`, `PlaceCommand`, and `UseCommand`. `Owner` enum: `MOVE`, `MINE`, `PLACE`, `USE`, `ATTACK`, `FOLLOW`, `SYSTEM`. `NavigationRequest` record: `owner`, `destinationSupplier`, `arrivalDistance`, `recalcDistance`, `maxNullPathRecalculations`, `onArrive`, `onCancel`, `onPathFailure`, `lockOnArrival` (optional — when non-null, `cleanup()` atomically calls `manager.lockForAction(botUuid, lockOnArrival)` before firing `onArrive`, eliminating the one-tick gap between nav-lock release and action-lock acquisition). Backward-compatible 8-arg constructor sets `lockOnArrival = null`. Public API: `navigate(fp, request)`, `cancel(uuid)`, `cancelAll()`, `cancelAll(owner)`, `isNavigating(uuid)`, `isNavigating(uuid, owner)`, `getOwner(uuid)`; exposed via `plugin.getPathfindingService()`. **`BotNavUtil`** — static utility class consolidating navigation/interaction helpers shared by `MineCommand` and `PlaceCommand`: `findStandLocation(World, SelectionBounds, tx, ty, tz)` (16-candidate walkable-adjacent search, prefers positions outside the selection), `faceToward(from, target)` (computes yaw+pitch toward a location), `isAtActionLocation(bot, loc)` (XZ ≤ 0.35 proximity check), `useStorageBlock(bot, block)` (NMS right-click via `gameMode.useItemOn`). `SelectionBounds` is an inner `@FunctionalInterface` (`boolean contains(int x, int y, int z)`) so callers pass `sel::contains` without coupling to command internals. **`StorageInteractionHelper`** — encapsulates the lock→open-container→transfer→unlock lifecycle shared by `MineCommand` (deposit bot→storage) and `PlaceCommand` (fetch storage→bot). Static entry point: `interact(fp, faceLoc, block, plugin, manager, transferFn, onFinally)`. Timing: bot locked + teleported on call; +1 tick container opened via NMS; +3 ticks transfer runs, inventory closed, action lock released, GUI refreshed, `onFinally` called. All error paths (bot offline, block no longer a container) also call `onFinally` so callers can clean up gating flags (e.g. `job.depositingToStorage = false`). **BotSwapAI rejoin lifecycle (v1.5.17 critical fix):** rejoin task is now registered AFTER `delete()` completes so `cancel(uuid)` never silently kills the rejoin timer — bots actually come back. New config: `swap.min-online` (floor that prevents swaps if removing one would drop below), `swap.retry-rejoin` (auto-retry failed rejoins, default `true`), `swap.retry-delay` (seconds before retry, default `60`). Bot identification on rejoin: same-name rejoins use `getByName()` (O(1) `nameIndex` lookup); random-name rejoins use UUID diff. Sub-package `fakeplayer/network/` contains the fake Netty layer: `FakeChannel` (discard channel used by `FakeConnection`), `FakeChannelPipeline` (no-op pipeline), `FakeConnection` (NMS `Connection` subclass — overrides all three `send()` variants as no-ops to prevent vanilla connection side-effects during `placeNewPlayer()` that previously caused a phantom "Anonymous User" / UUID-0 ghost entry; `isConnected()` always returns `true`; injects `FakeChannel` and calls `Connection.configureSerialization()` on the no-op pipeline to satisfy NMS internal checks), and `FakeServerGamePacketListenerImpl` (no-op SGPL subclass that discards all outbound packets except knockback). Note: `BotHeadAI` has been removed; head-AI functionality (player tracking, look-range, turn-speed) is now driven by config keys `head-ai.*` and executed inside `FakePlayerManager`.
- `src/main/java/me/bill/fakePlayerPlugin/messaging/` — proxy communication (`VelocityChannel`): registers `fpp:main` plugin-messaging channel; wraps outbound messages in a BungeeCord `Forward ALL` envelope so Velocity/BungeeCord re-delivers to every backend. Subchannels: `BOT_SPAWN`, `BOT_DESPAWN`, `BOT_UPDATE` (display-name sync — wire: `[msgId][serverId][uuid][newDisplayName]`; sent by `broadcastBotDisplayNameUpdate(fp)` when LP refreshes a bot's display name), `CHAT`, `ALERT`, `JOIN`, `LEAVE`, `SYNC`. Echo suppression via per-message unique IDs.
- `src/main/java/me/bill/fakePlayerPlugin/sync/` — proxy config sync (`ConfigSyncManager`): push/pull config files to/from MySQL in NETWORK mode. Supports `DISABLED`, `MANUAL`, `AUTO_PULL`, `AUTO_PUSH` modes. Server-specific keys (`database.server-id`, `database.mysql.*`, `debug`) are never synced.
- `src/main/java/me/bill/fakePlayerPlugin/database/` — session history and analytics (`DatabaseManager`, `BotRecord`). **Schema version 19** — persistent tables include `fpp_bot_sessions`, `fpp_active_bots`, `fpp_skin_cache`, `fpp_despawn_snapshots`, `fpp_sleeping_bots`, `fpp_bot_identities`, `fpp_bot_tasks`, and `fpp_meta`. Use `DatabaseManager.getSchemaVersion()` as the source of truth.
- `src/main/java/me/bill/fakePlayerPlugin/lang/` — message rendering (`Lang`).
- `src/main/java/me/bill/fakePlayerPlugin/permission/` — centralised permission registry (`Perm`).
- `src/main/java/me/bill/fakePlayerPlugin/ai/` — AI conversation system: `AIProvider` (interface with `ChatMessage` record), `AIProviderRegistry` (selects first available provider; `isAvailable()`, `getActiveProvider()`, `generateResponse()`), `BotConversationManager` (per-bot-player conversation history, DM handling, public-chat reactions, personality resolution, rate limiting; `handleDirectMessage`, `generatePublicChatReaction`, `clearAll()`), `PersonalityRepository` (loads `.txt` files from `plugins/FakePlayerPlugin/personalities/`; `get(name)`, `has(name)`, `getNames()`, `size()`, `reload()`). `size()` is used during startup debug logging. Seven concrete providers: `OpenAIProvider`, `AnthropicProvider`, `GroqProvider`, `GoogleGeminiProvider`, `OllamaProvider`, `CopilotProvider`, `CustomOpenAIProvider`. API keys live in `plugins/FakePlayerPlugin/secrets.yml` (managed by `config/SecretsConfig.java`); template extracted from JAR on first run. Accessed via `plugin.getAIProviderRegistry()`, `plugin.getBotConversationManager()`, `plugin.getPersonalityRepository()`.
- `config/SecretsConfig.java` — manages `plugins/FakePlayerPlugin/secrets.yml`; exposes API key accessors for all seven AI providers (`openaiApiKey()`, `groqApiKey()`, etc.); `init(plugin)` copies template from JAR on first run; `reload()`. **Never commit real API keys** — the template file is the only version-controlled copy.

2) Where to start (entry points)
- `FakePlayerPlugin.java` — plugin lifecycle (`onEnable` / `onDisable`) and orchestration.
- `src/main/java/me/bill/fakePlayerPlugin/command/` — command classes (`SpawnCommand`, `DeleteCommand` (registers as `despawn`; subcommands: `<name>`, `all` (requires `Perm.DELETE_ALL`), `--random [num]` (despawn N random bots), `--num <num>` (despawn N oldest bots); **v1.6.3:** bulk operations `all`, `--random`, `--num` are blocked while `manager.isRestorationInProgress()` returns `true` — sends the `delete-restore-in-progress` lang key; single-bot `<name>` despawn is unaffected), `ListCommand`, `HelpCommand`, `InfoCommand`, `ChatCommand`, `SwapCommand`, `ReloadCommand`, `FreezeCommand`, `StatsCommand`, `MigrateCommand`, `TpCommand`, `TphCommand`, `LpInfoCommand`, `RankCommand`, `AlertCommand`, `SyncCommand`, `PeaksCommand`, `SettingCommand`, `InventoryCommand`, `MoveCommand`, `XpCommand`, `CmdCommand`, `MineCommand`, `FindCommand`, `UseCommand`, `AttackCommand`, `WaypointCommand`, `BadwordCommand`, `RenameCommand`, `PersonalityCommand`, `PlaceCommand`, `StorageCommand`, `FollowCommand`, `SleepCommand`, `StopCommand`, `SetOwnerCommand`, `BotSelectCommand`, `BotGroupCommand`) plus `WaypointStore` (named route persistence used by `/fpp waypoint` and `/fpp move --wp`), `StorageStore` (per-bot supply-container registry, stored in `data/bot-data.yml` under `storages.by-bot`), `BotGroupStore` (per-player bot groups, `default`/`owned` resolve to active bots owned by that player), `MineSelectionStore` (area selection persistence for `/fpp mine`, `data/mine-selections.yml`). Stripmine and farming automation now live in the addon plugin (`fpp-addon/`) via `/fpp mine --stripmine` command extension and `/fpp farm` addon command; ping, auto-equipment, sprint-jump, and WorldEdit placement helpers also moved there. The companion sample addon lives in `hello-world-addon/`.
- `command/FppCommand.java` — interface every sub-command implements (`getName`, `getAliases` (default `emptyList()` — used e.g. by `InventoryCommand` for alias `"inv"`), `getUsage`, `getDescription`, `getPermission`, `canUse`, `execute`, `tabComplete`).
- `config/Config.java`, `util/ConfigMigrator.java` — configuration pattern and automatic migration.
- `fakeplayer/FakePlayerManager.java`, `fakeplayer/BotPersistence.java` — bot lifecycle, persistence and restore. Key new helpers: `requestNavJump(UUID)` — sets a 5-tick jump-hold for navigation (used by `MoveCommand` to prevent the swim-AI from cancelling jumps before physics fires); `clearNavJump(UUID)` — clears the pending jump on despawn. **Action lock API (renamed from mining lock):** `lockForAction(UUID, Location)` — freezes the bot at the given position and suppresses head-AI for the duration of a block interaction (mining, storage deposit/fetch, right-click use); `unlockAction(UUID)` — releases the lock; `isActionLocked(UUID)` — query; `updateActionLockRotation(UUID, float yaw, float pitch)` — updates the bot's stored yaw/pitch while locked (used by `AttackCommand` mob mode for smooth rotation without releasing the lock). Backed by `actionLockedBots` (`ConcurrentHashMap<UUID, Location>`). Previously named `lockForMining`/`unlockMining`/`isMiningLocked` — all old names have been removed. **Navigation lock API (separate from action lock):** `lockForNavigation(UUID)` / `unlockNavigation(UUID)` / `isNavigationLocked(UUID)` — backed by `navLockedBots` (`ConcurrentHashSet<UUID>`). Suppresses head-AI rotation while a bot is actively navigating (PathfindingService calls `lockForNavigation` on `navigate()` start and `unlockNavigation` on `cancel()`). Does NOT freeze bot position — that is the action lock's job. **`bodyTransitionBots`** (Set<UUID>): `isBodyTransitioning(UUID)` — `true` while a bot's NMS body is being swapped (e.g. during world-change re-spawn); prevents duplicate entity registration. **`despawningBotIds`** (`ConcurrentHashMap<UUID, String>`): value is the bot's display name at despawn time; `isDespawning(UUID)` / `getDespawningDisplayName(UUID)` allow event listeners to identify bots being removed and retrieve their display name during the removal window. **`refreshCleanNamePool()`** — pre-filters all names from `bot-names.yml` through the badword filter once; the result is cached in `cleanNamePool` and used by `generateName()` instead of running the expensive regex passes inline. Call this after any badword reload (already done automatically on startup and by `ReloadCommand`). **Entity ID index (v1.6.3):** `entityIdIndex` (`ConcurrentHashMap<Integer, FakePlayer>`) — secondary O(1) lookup by NMS entity ID, maintained at all add/remove/world-change sites. `getByEntity(Entity entity)` checks this first, then falls back to PDC name tag for world-change recovery (and self-heals the index). Public helpers: `removeFromEntityIndex(int entityId)`, `registerEntityIndex(int entityId, FakePlayer fp)` — used by `FakePlayerEntityListener` and `PlayerWorldChangeListener` to keep the index accurate.
- `fakeplayer/BotPathfinder.java` — server-side A* grid pathfinder for bot navigation (used by `MoveCommand`). Supports 8 `MoveType` actions: `WALK`, `ASCEND`, `DESCEND`, `PARKOUR`, `BREAK`, `PLACE`, `PILLAR`, `SWIM`. Feature flags passed via `PathOptions` record (`parkour`, `breakBlocks`, `placeBlocks`, `avoidWater`, `avoidLava`); all default `false`. Key limits are config-driven (`pathfinding.max-range`, `max-nodes`, `max-nodes-extended`, `max-fall`). Detour system: when A* returns no path, `findDetourGoal()` samples up to `Config.pathfindingDetourAttempts()` random offsets within `Config.pathfindingDetourRadius()` blocks and retries — prevents bots getting permanently stuck at obstacles. Unloaded-chunk handling: unloaded blocks are always treated as passable (hardcoded `true`) so the pathfinder can plan through unloaded regions. Public entry points: `BotPathfinder.findPathMoves(world, sx,sy,sz, tx,ty,tz, opts)` → `List<Move>`; `BotPathfinder.findPath(...)` → `List<Pos>` (backward-compat, strips action metadata). `walkable(world, x, y, z)` and `passable(world, x, y, z)` are public world-query helpers.
- `command/InventoryCommand.java` — `/fpp inventory <bot>` (alias `inv`). Opens a 54-slot double-chest GUI displaying the bot's full inventory (main storage rows 1-3, hotbar row 4, label bar row 5, equipment+offhand row 6). Implements both `FppCommand` and `Listener`. Equipment slots enforce type restrictions (boots/leggings/chestplate-or-elytra/helmet/offhand). Right-clicking a bot entity directly opens the GUI without a command (unless shift-held, which opens `BotSettingGui` instead). `openGui(Player, FakePlayer)` is the public entry point. Constructor: `new InventoryCommand(fakePlayerManager, plugin, botSettingGui)`. Registered as a listener in `onEnable` immediately after command registration. Permission: `Perm.INVENTORY = "fpp.inventory"`.
- `command/MoveCommand.java` — `/fpp move <bot|all> --to <player>  |  <bot|all> --coords <x> <y> <z>  |  <bot|all> --wp <route> [--random]  |  <bot|all> --roam [x,y,z] [radius]  |  <bot|all> --stop`. **`--coords` flag:** parses three doubles (supports `~` relative notation against sender's location), builds a fixed `Location`, and navigates via `startNavigationToLocation()` (`Owner.MOVE`). Lang keys: `move-coords-usage`, `move-coords-invalid`, `move-coords-navigating`. Navigates a bot (or all bots) to an online player using `PathfindingService`, sends them on a looping waypoint patrol, or starts autonomous random wandering around a center point. The `--to <player>` flag is now the canonical form for player-follow navigation (the old positional `<bot> <player>` syntax still works as a backward-compat fallback). **`--roam` mode:** bot wanders continuously within a configurable radius (3–500 blocks) around a fixed center; if no coordinates are given the bot's current location is used. Internal state: `roamCenters` / `roamRadii` (`ConcurrentHashMap<UUID, …>`) and `roamPauseTasks` / `roamLookTasks` for idle scheduling between steps. New public API: `isRoaming(UUID)`, `getRoamCenter(UUID)`, `getRoamRadius(UUID)`, `resumeRoaming(FakePlayer, Location center, double radius)` (called by `BotPersistence` after restart). Roam state is persisted to `data/bot-tasks.yml` YAML only (not in `fpp_bot_tasks` DB table — roam is not a DB `BotTaskRow` task type). Recalculates path when target moves >3.5 blocks or every 60 ticks. Calls `manager.requestNavJump(UUID)` for ascend/parkour/stuck-recovery jumps. Stuck detection (8 ticks without movement) forces jump + recalc. Public `cleanupBot(UUID)` method clears the bot's entries from `activeRouteNames`/`activeRandomFlags`/`randomWpOrder`/`roamCenters`/`roamRadii`/`roamPauseTasks`/`roamLookTasks` maps to prevent memory leaks on bot despawn — called by `FakePlayerManager.delete()`. `cancelAll()` is a public method but is NOT called in `onDisable`. Additional public API: `getActiveRouteForBot(UUID)` → route name or `null`; `isRandomPatrolForBot(UUID)` → random-order flag; `resumePatrol(Player, List<Location>, boolean random, String routeName)` — restores patrol state after server restart (called by `BotPersistence`). New lang keys: `move-roam-started` (`{name}`, `{x}`, `{y}`, `{z}`, `{radius}`), `move-all-roam-started` (`{count}`, `{radius}`, `{skipped}`), `move-roam-invalid-coords`, `move-roam-invalid-radius`, `move-roam-usage`. Permission: `Perm.MOVE = "fpp.move"`.
- `command/XpCommand.java` — `/fpp xp <bot>`. Transfers all XP (`getTotalExperience()`) from a bot to the command sender (player only; console gets nothing but XP is still cleared). Clears bot's total XP, level, and progress. Imposes a 30-second post-collection cooldown on the bot's XP pickup (`BotXpPickupListener` reads `plugin.getXpCommand().isOnXpCooldown(uuid)`). Exposed via `plugin.getXpCommand()`. Permission: `Perm.USER_XP = "fpp.xp"` (user-tier, included in `fpp.use`).
- `command/CmdCommand.java` — `/fpp cmd <bot> <command...>` (direct execution), `/fpp cmd <bot> --add <command...>` (store right-click command), `/fpp cmd <bot> --clear` (remove stored command), `/fpp cmd <bot> --show` (display stored command). Uses `Bukkit.dispatchCommand()` which bypasses `PlayerCommandPreprocessEvent` so `BotCommandBlocker` does not interfere. Right-click behavior handled in `InventoryCommand.onRightClickBot()` — bots with stored commands execute them instead of opening inventory GUI. Permission: `Perm.CMD = "fpp.cmd.admin"`; `canUse()` checks `Perm.hasAny(Perm.CMD, Perm.CMD_LEGACY)` so holders of the legacy `fpp.cmd` node also qualify.
- `command/MineCommand.java` — `/fpp mine <bot>` (continuous mining), `/fpp mine <bot> once` (mine one block), `/fpp mine <bot> stop` (stop mining), `/fpp mine stop` (stop all bots). Bots continuously break blocks they're looking at using NMS `handleBlockBreakAction()` and `destroyBlockProgress()` packets. Creative mode: instant break with 5-tick cooldown. Survival mode: progressive mining with destroy-speed accumulation. Mining state tracked per-bot (`MiningState` — `currentPos`, `progress`, `freeze`, `once`). Area-selection mode: `/fpp mine <bot> --pos1`, `--pos2`, `--start`, `--status`, `--stop` — selects a cuboid region and mines it continuously; selections persist across restarts via `MineSelectionStore` (`data/mine-selections.yml`) and auto-resume. Finder mode is also exposed through mining: `/fpp mine <bot|all> --find <block> [count] [--radius <n>] [--prefer-visible]`, delegated to `FindCommand`. Constructor: `new MineCommand(plugin, fakePlayerManager, storageStore, mineSelectionStore, pathfindingService)`. Restocks from nearest enabled `StorageStore` container when inventory fills in area mode. Public API: `stopMining(UUID)`, `stopAll()`. Permission: `Perm.MINE = "fpp.mine"`. Stripmine now lives in `fpp-addon/` as a `/fpp mine --stripmine` command extension.
- `command/RenameCommand.java` — `/fpp rename <oldname> <newname>`. Renames an active bot by fully despawning the old entity and respawning it under the new name. Rename lifecycle is now delegated to `BotRenameHelper` (shared with `BotSettingGui`). Preserves all bot state via `BotSnapshot`: inventory (main/armor/offhand, deep-cloned), XP (level + progress + total), chat settings (enabled flag + activity tier), frozen state, LuckPerms group, `aiPersonality`, and original spawner info. **Rename lifecycle:** (1) snapshot captured from old FakePlayer, (2) `manager.markRenaming(oldUuid)` called to suppress join/leave broadcasts, (3) `manager.delete(oldName)` fires, (4) 12 ticks later `manager.spawn(loc, 1, null, newName, true, botType)` called, (5) new bot UUID also marked renaming, (6) `scheduleRestore` polls every 5 ticks until NMS entity ready (up to 200 ticks), (7) snapshot applied, LP group restored 5 ticks later, (8) both UUIDs unmarked, (9) `bot-rename` Lang broadcast sent to all players. 4) Common tasks & minimal steps (how to implement typical changes)
- Add a new command:
    - Create `src/main/java/me/bill/fakePlayerPlugin/command/MyCommand.java` implementing the command interface used by `CommandManager`.
    - Register it in `FakePlayerPlugin.onEnable()` with `commandManager.register(new MyCommand(...))`.
    - No manual help update is needed — `HelpCommand` reads live from `CommandManager`.
    - Ensure `plugin.yml` contains the `fpp` command or sub-command docs (command parsing done by `CommandManager`).
- Add a config option:
    - Add default to `src/main/resources/config.yml`.
    - If needed, update `Config` accessor and `ConfigMigrator` to preserve old configs.
    - New options should be validated in `util/ConfigValidator.java` if they can be misconfigured.
- Add a language message:
    - Add the key and default text to `src/main/resources/language/en.yml`.
    - Access it in code with `Lang.get("your-key")` or `Lang.raw("your-key", "placeholder", "value")`.
    - The file is hot-reloadable — `/fpp reload` picks up changes without restart.
- Add a permission node:
    - Declare a new `public static final String` constant in `permission/Perm.java` (e.g. `public static final String MY_FEATURE = "fpp.myfeature";`).
    - Add a full entry under `permissions:` in `plugin.yml` with description and default value (e.g. `op`, `true`, or `false`).

    - Use the constant in your command's `getPermission()` and `canUse()` methods: `Perm.has(sender, Perm.MY_FEATURE)`.
    - **Critical:** LuckPerms reads `plugin.yml` at startup for tab-completion — permissions missing from `plugin.yml` won't appear in `/lp permission set fpp.<tab>`.
    - Test: restart server, run `/lp user <name> permission set fpp.` and verify your new permission appears in tab-completion.

        - If changing skin behaviour or adding skins: ensure `SkinRepository`/`SkinFetcher` are respected — add skin files to `src/main/resources/skins/` for defaults or `plugins/FakePlayerPlugin/skins/` at runtime and call `/fpp reload` to refresh. Full `/fpp migrate` subcommands: `backup` (create manual backup), `backups` (list stored backups), `status` (config version + file sync health + DB stats), `config` (re-run config migration chain), `lang` / `names` / `messages` (force-sync the respective YAML file from JAR), `db merge [file]` / `db export` / `db tomysql`. Set `database.enabled: false` in config to disable all DB I/O.
- Tab list header/footer and update checker: `TabListManager.start()` and `UpdateChecker.check(this)` are run during `onEnable()` — useful when diagnosing missing tab headers or unexpected update-checker output.

7) Integration & soft-dependencies
- PlaceholderAPI: `util.FppPlaceholderExpansion` registered if plugin present. **Full placeholder list (29+ total)** — server-wide: `%fpp_count%` (bot count — local + remote in NETWORK mode), `%fpp_local_count%` (this server's bots only), `%fpp_network_count%` (remote servers' bots only), `%fpp_max%` (global cap, `∞` if 0), `%fpp_real%` (non-bot players online), `%fpp_total%` (real + bots, network-wide), `%fpp_online%` (alias for `%fpp_total%`), `%fpp_frozen%`, `%fpp_names%` (comma-joined display names — local + remote in NETWORK mode), `%fpp_network_names%` (remote-server bot names only), `%fpp_chat%`, `%fpp_swap%`, `%fpp_skin%`, `%fpp_body%`, `%fpp_pushable%`, `%fpp_damageable%`, `%fpp_tab%`, `%fpp_max_health%`, `%fpp_persistence%`, `%fpp_version%`. Network/Proxy: `%fpp_network%` (on/off for NETWORK mode), `%fpp_server_id%`, `%fpp_spawn_cooldown%`. Per-world: `%fpp_count_<world>%`, `%fpp_real_<world>%`, `%fpp_total_<world>%`. Player-relative: `%fpp_user_count%` (bots owned by that player), `%fpp_user_max%` (personal limit), `%fpp_user_names%` (comma-joined names of that player's bots). `%fpp_real%` = `Bukkit.getOnlinePlayers().size() - localBots` — bots go through `placeNewPlayer()` so they appear in `getOnlinePlayers()`; FPP subtracts local bot count to get the true real-player count. `%fpp_total%` = `real + bots`.
- LuckPerms: `LuckPermsHelper.subscribeLpEvents(plugin, manager)` is called directly in `onEnable` (not deferred) for auto-prefix/suffix refresh via `UserDataRecalculateEvent`.
- WorldGuard: soft-depend declared in `plugin.yml`. `WorldGuardHelper.isPvpAllowed(location)` cancels player-sourced PvP damage to bots in no-PvP regions (handled in `FakePlayerEntityListener` at `LOWEST` priority). `worldGuardAvailable` is cached in `onEnable` via a pure Bukkit plugin check — **never call `WorldGuardHelper` directly without first checking `plugin.isWorldGuardAvailable()`** (same ClassLoader guard as LuckPerms).
- Vercel / Node API: `UpdateChecker` falls back from Modrinth to `https://fake-player-plugin.vercel.app`; `frontend/api/check-update.js` and `frontend/api/status.js` intentionally emit Java-compatible keys such as `remoteVersion`, `version`, `version_number`, and `downloadUrl`.
- Metrics: `util/FppMetrics` wraps FastStats and is conditional via `config.yml` flag (`Config.metricsEnabled()`).
